# Phase 16 Commercial Language And Positioning Posture

## Positioning posture

- Use `launch operating system` as the core product frame.
- Use `self-serve first` to describe the default commercial motion.
- Use `enterprise review` or `enterprise path`, not `contact us for custom stuff`.
- Use `talk to sales` or `request demo` for high-intent buyer actions.
- Use `buyer guides` for evaluation docs, not `marketing resources`.

## Pricing language rules

- Free should read as a real tier, not a fake teaser.
- Starter should read as the first serious live team package.
- Growth should read as the operator tier.
- Enterprise should read as a control and onboarding posture, not a vague upsell bucket.

## Buyer copy posture

- Commercial copy should explain why the product is broader than quests or airdrops alone.
- Package differences should be framed through capacity and operating depth, not random feature fragmentation.
- CTA copy should make it clear when a team can self-serve and when a human path makes more sense.
- Trust, pricing and enterprise intake should feel like one coherent path.

## Internal commercial posture

- Leads should be described by state and intent, not by gut feel.
- `new`, `qualified`, `engaged`, `evaluation`, `converted`, `cooling_off` and `lost` are the preferred lead states.
- Notes should capture buyer concerns and next steps, not generic summaries.
- Follow-up tasks should always describe a concrete next move and due posture.

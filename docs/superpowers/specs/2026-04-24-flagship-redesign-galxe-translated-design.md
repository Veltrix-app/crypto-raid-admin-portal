# Flagship Redesign: Galxe-Translated Premium UI

Date: 2026-04-24
Status: Ready for review
Scope: Homepage, pricing, and portal overview redesign direction

## 1. Goal

Move Veltrix from a "refined existing UI" into a visibly stronger premium product identity.

The redesign should solve a more ambitious problem than the quiet-system pass:

- make the product feel modern, premium, and unmistakably intentional
- make visitors understand the product faster
- make the portal feel more like a high-end command system
- bring webapp/public and portal into one clear design family
- capture what works well in Galxe's visual clarity without turning Veltrix into a copy

This is not a generic SaaS polish pass.

This is a flagship redesign of the three most important screens that establish the visual truth for the rest of the product:

- `/`
- `/pricing`
- `/overview`

## 2. Problem Statement

Veltrix is strong in product breadth, but the visual system still does not fully communicate that strength.

The current UI has improved hierarchy, but the next gap is no longer just "too many cards."

The bigger issue is that the product still does not yet feel as:

- premium
- calm
- focused
- memorable

as the strongest competitors.

The user expectation benchmark is now set closer to Galxe:

- clearer first impression
- better scanability
- calmer section rhythm
- more premium dark-mode polish
- stronger visual focus

Veltrix should not imitate Galxe literally.

It should translate Galxe's strengths into a Veltrix-native product language:

- less marketplace
- more launch operating system
- less noisy campaign wall
- more premium command surface

## 3. Design Direction

### 3.1 Chosen Direction

The redesign direction is:

- dark-first
- premium
- clear
- controlled web3 energy
- Galxe-translated, not Galxe-copied

### 3.2 Portal vs Public Relationship

Portal and public surfaces should belong to the same design family.

They should not feel like different products.

But they should differ in intensity:

- public/webapp: more expressive, more atmospheric, more premium showroom energy
- portal: calmer, more functional, more command-center clarity

The family should be the same.
The emotional amplitude should be different.

### 3.3 Emotional Target

Veltrix should feel like:

- a premium launch operating system
- web3-native without becoming noisy
- modern and high-trust
- clear enough for buyers and operators to instantly orient themselves

It should not feel like:

- a flat enterprise dashboard
- a generic SaaS template
- an over-styled degen UI
- a campaign marketplace clone

## 4. Approach Options

### Option A: Hero-first facelift

Only redesign the top sections of the three flagship screens.

Pros:

- fast
- visually noticeable
- lower implementation time

Cons:

- the rest of each page would still feel old
- creates a split between "wow hero" and "old body"
- too weak for the level of redesign now desired

### Option B: Full-screen flagship redesign

Fully redesign homepage, pricing, and portal overview as complete screens.

Pros:

- strongest reset
- creates a real visual standard
- produces flagship references for the rest of the system

Cons:

- more design effort
- requires stronger discipline to keep consistency

### Option C: Design system first

Spend the next pass on tokens, primitives, and component language first.

Pros:

- methodical
- technically clean

Cons:

- too abstract for the current need
- delays the visible transformation
- does not create immediate momentum

### Recommendation

Build Option B.

Redesign the three flagship screens fully, and use those screens as the visual truth for the next waves.

That gives Veltrix a visible leap in quality without requiring an all-at-once full-platform rewrite.

## 5. Core Design Principles

### 5.1 Clarity First

Every flagship page must make it obvious within a few seconds:

- what this page is
- why it matters
- what the user should do next

This remains the primary UX rule even inside a more expressive redesign.

### 5.2 Premium Darkness, Not Flat Darkness

The base should remain dark-first, but not as a flat black canvas.

Use:

- layered dark surfaces
- soft tonal transitions
- subtle gradients
- restrained glow

Avoid:

- empty black slabs
- overly neon panels
- overly sharp border-heavy darkness

### 5.3 Web3 Energy With Restraint

Veltrix should carry web3 energy, but in a controlled way.

Use energy through:

- atmospheric gradients
- accent glow on focal elements
- sharper typography moments
- vivid accent color on CTA and status emphasis

Avoid:

- visual chaos
- too many accent colors
- decorative energy on low-priority content

### 5.4 One Visual Story Per Viewport

Each screen above the fold should have one dominant story, not several competing stories.

That means:

- one dominant headline
- one dominant content cluster
- one dominant action lane

### 5.5 Same Family, Different Intensity

Public and portal must feel related.

But:

- public gets more mood and visual drama
- portal gets more structure and faster scanability

## 6. Visual Language

### 6.1 Palette Direction

Primary base:

- deep charcoal
- blue-black
- smoked graphite

Accent family:

- acid-lime / electric-lime for primary emphasis
- cyan / cool aqua for secondary energy

Support tones:

- slate
- soft silver
- muted off-white text

Meaning mapping:

- lime = primary momentum, CTA, success-forward emphasis
- cyan = system energy, navigation or highlight support
- amber = caution
- rose = failure or risk

### 6.2 Surface Language

Use:

- large, quiet blocks
- softer containers
- tonal surface shifts
- minimal but intentional borders

Avoid:

- hard card grids everywhere
- card-on-card-on-card composition
- outlines used as decoration

The visual separation should come more from:

- spacing
- typography
- layering
- tonal contrast

than from repeated boxes.

### 6.3 Typography Direction

Typography should do more of the hierarchy work.

Use:

- bold, high-contrast display headlines
- strong section titles
- compact explanatory copy
- restrained overlines/eyebrows

Avoid:

- too many tiny uppercase labels
- overly compressed dense meta text
- equal visual weight across every module

### 6.4 Motion

Motion should be subtle but meaningful.

Use:

- gentle page-load reveal
- soft hover elevation
- restrained accent shimmer or glow on focal CTA

Avoid:

- constant animation
- generic micro-motion everywhere
- decorative motion on secondary surfaces

## 7. Flagship Screen 1: Homepage

### 7.1 Objective

The homepage must communicate:

- what Veltrix is
- why it matters
- why it feels premium
- what the next step is

### 7.2 Structural Pattern

The homepage should read in this order:

1. flagship hero
2. system story
3. product operating model
4. commercial confidence
5. trust and enterprise path
6. final CTA

### 7.3 Hero

The hero should include:

- large flagship headline
- short, sharp supporting copy
- two clear CTAs:
  - `Start now`
  - `Talk to sales`
- one premium visual product composition

The visual should not be a random illustration.

It should feel like a stylized product world:

- partial product preview
- layered interface fragments
- launch/operations atmosphere

### 7.4 Middle Sections

Below the hero, use 3 to 4 large story sections:

- Launch
- Operate
- Grow
- Control

Each section should have:

- a headline
- a short explanation
- a visual composition or mock surface
- one route deeper into the product story

### 7.5 Final Impression

The homepage should feel like a premium product ecosystem, not a landing-page checklist.

## 8. Flagship Screen 2: Pricing

### 8.1 Objective

Pricing must feel:

- trustworthy
- simple to scan
- commercially mature
- easy to act on

### 8.2 Structural Pattern

The pricing page should read in this order:

1. pricing hero
2. plan cards
3. plan guidance
4. trust and buyer confidence
5. enterprise CTA

### 8.3 Hero

The top section should include:

- strong pricing headline
- short explanation of the pricing philosophy
- two clear routes:
  - `Start now`
  - `Talk to sales`

### 8.4 Plan Presentation

Avoid a traditional cold pricing table as the dominant structure.

Instead:

- use four high-quality pricing cards
- give them clear hierarchy
- visually center Growth as the commercial sweet spot
- separate Enterprise as a higher-touch option

Each plan card should clearly show:

- plan name
- price
- ideal user
- key limits
- key unlocks
- primary CTA

### 8.5 Decision Help

After the cards, add a layer that explains:

- who Free is for
- when to choose Starter
- when Growth becomes the right answer
- when Enterprise should go to human contact

This makes the page feel more guided and reduces pricing ambiguity.

## 9. Flagship Screen 3: Portal Overview

### 9.1 Objective

The overview must feel like:

- a command center
- a premium operator surface
- a calm launch-control page

### 9.2 Structural Pattern

The page should read in this order:

1. overview hero
2. `Now`
3. `Next`
4. `Watch`
5. secondary context

### 9.3 Top Zone

The top zone should include:

- workspace-aware heading
- short status line
- one primary action
- one secondary action
- very compact high-value signals

This top zone must answer immediately:

- what is happening
- what matters most
- what the operator should do next

### 9.4 Main Body

The main body should be grouped around three high-level lanes:

#### Now

Current active state:

- live campaigns
- urgent launch signals
- immediate operational pressure

#### Next

Clear next actions:

- launch setup
- unresolved blockers
- team, project, campaign, billing, or support actions

This should be the most directive section.

#### Watch

Lower-weight monitoring context:

- health
- risk
- support
- trust
- billing posture

### 9.5 Result

The portal overview should feel much more like "your launch command center" and much less like "a lot of modules."

## 10. What To Borrow From Galxe

Borrow:

- dark-first polish
- calmer card rhythm
- strong section hierarchy
- clear visual focus above the fold
- better scanability
- stronger premium feel

Do not borrow:

- exact page composition
- marketplace-heavy behavior
- excessive small-card repetition
- consumer-explorer feel as the product default

Veltrix should feel more like:

- a launch operating system
- not a quest marketplace

## 11. Implementation Strategy

This redesign should be built as a new flagship visual standard, then used to guide the next rollout waves.

Recommended order:

1. redesign homepage
2. redesign pricing
3. redesign portal overview
4. validate the family match between public and portal
5. use the resulting language to expand into the next UI waves

## 12. Success Criteria

The redesign is successful if:

- the homepage feels premium and immediately understandable
- pricing becomes clearer and more trustworthy
- portal overview becomes more directional and high-end
- public and portal visibly belong to the same family
- Veltrix feels closer to a premium product benchmark like Galxe in polish and clarity
- Veltrix still feels like Veltrix, not like a clone

## 13. Out Of Scope For This Spec

This design spec does not yet cover:

- full implementation details
- component-level code changes
- every portal screen
- full docs redesign
- full builder redesign beyond future extrapolation from the flagship screens

Those come next in the implementation plan.
